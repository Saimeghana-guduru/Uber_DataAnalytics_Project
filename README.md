# 🚖 Uber Data Analytics Interactive Dashboard

An interactive, high-fidelity business intelligence dashboard built using **Power BI Desktop** that simulates a ride-hailing operations center. It provides end-to-end performance visibility into ride requests, revenue streams, cancellations, driver/customer metrics, and regional booking hot spots.

---

## 📊 Visual Features & Interactivity

The dashboard utilizes advanced, custom-built navigation systems to achieve a seamless, fluid web-app feel entirely offline:

*   **Dynamic 3D Vehicle Fleet Viewers (Bookmarks Engine):** Tapping the thumbnail deck in the bottom-left corner dynamically switches a prominent center-left panel to display a detailed 3D render of the selected asset class (`Bike`, `Auto`, or `Cab`). 
*   **Fully Synced Cross-Filtering:** Selecting a vehicle asset type doesn't just change the graphics—it simultaneously filters all structural matrices, line graphs, and top KPI cards to isolate metrics exclusively for that category.

---

## 📂 Dataset Overview

The dashboard is powered by the included `ride_bookings.csv` file, consisting of **150,000 transaction records** capturing city ride metrics.

### Key Data Columns Utilized:
*   **Core Identifiers:** `Booking ID`, `Customer ID`, `Date`, `Time`
*   **Operational Context:** `Vehicle Type`, `Pickup Location`, `Drop Location`, `Booking Status` (Completed, Cancelled by Driver, No Driver Found, Cancelled by Customer, Incomplete)
*   **Financials & Logistical Metrics:** `Booking Value` (Revenue), `Ride Distance`
*   **Qualitative Feedback:** `Driver Ratings`, `Customer Rating`

## 🛠️ Infrastructure & Data Modeling (DAX)

To maintain high performance and prevent cross-filtering fragmentation, a proper **Star Schema** data model was implemented:

1.  **Vehicle Group Mapping Column:** Programmed to group seven granular vehicular categories (`eBike`, `Go Sedan`, `Auto`, etc.) into three operational buckets:
    ```dax
    Vehicle_Group = 
    SWITCH(
        TRUE(),
        'ncr_ride_bookings'[Vehicle Type] = "Auto", "Auto",
        'ncr_ride_bookings'[Vehicle Type] IN {"Bike", "eBike"}, "Bike",
        "Cab"
    )
    ```

### Core Analytical Measures Used:

#### Key Performance Indicators (KPIs) & Averages:
*   **Completed Bookings:** Calculates total successful rides.
    ```dax
    Completed Bookings = CALCULATE(COUNT('ride_bookings'[Booking ID]), 'ride_bookings'[Booking Status] = "Completed")
    ```
*   **Lost Bookings:** Tracks all operational drop-offs (cancellations and unfulfilled matches).
    ```dax
    Lost Bookings = CALCULATE(COUNT('ride_bookings'[Booking ID]), 'ride_bookings'[Booking Status] <> "Completed")
    ```
*   **Total Revenue:** Gross value generated by rides.
    ```dax
    Total Revenue = SUM('ride_bookings'[Booking Value])
    ```
*   **Avg Distance:** Mean distance covered per ride trip.
    ```dax
    Avg Distance = AVERAGE('ride_bookings'[Ride Distance])
    ```
*   **Avg Customer Rating:** Average rating given *by* customers.
    ```dax
    Avg Customer Rating = AVERAGE('ride_bookings'[Customer Rating])
    ```
*   **Avg Driver Rating:** Average rating given *to* drivers.
    ```dax
    Avg Driver Rating = AVERAGE('ride_bookings'[Driver Ratings])
    ```

#### High-Frequency Location Metrics:
Rather than hardcoding specific values, the dashboard dynamically extracts the highest-frequency logistics nodes using Power BI’s native filtering engine:

*   **Top Drop Location & Count:**
    ```dax
    Top Pickup Location = TOPN(
    1, 
    VALUES('ride_bookings'[Pickup Location]), 
    CALCULATE(COUNTA('ride_bookings'[Pickup Location])))
    ```
    ```dax
    Top Pickup Count = MAXX(
    TOPN(
        1, 
        VALUES('ride_bookings'[Pickup Location]), 
        CALCULATE(COUNTA('ride_bookings'[Pickup Location]))
    ),
    CALCULATE(COUNTA('ride_bookings'[Pickup Location])))
    ```
    
*   **Top Drop Location & Count:**
    ```dax
    Top Pickup Location = TOPN(
    1, 
    VALUES('ride_bookings'[Drop Location]), 
    CALCULATE(COUNTA('ride_bookings'[Drop Location])))
    ```
    ```dax
    Top Pickup Count = MAXX(
    TOPN(
        1, 
        VALUES('ride_bookings'[Drop Location]), 
        CALCULATE(COUNTA('ride_bookings'[Drop Location]))
    ),
    CALCULATE(COUNTA('ride_bookings'[Drop Location])))
    ```

## 🚀 How to Run the Project Locally (No Account Required)

You do **not** need a paid or enterprise Power BI Service account to interact with this project. It is fully operational offline via Power BI Desktop.

1.  Clone this repository to your local machine:
    ```bash
    git clone [https://github.com/Saimeghana-guduru/Uber_DataAnalytics_Projec.git](https://github.com/YOUR_USERNAME/YOUR_REPOSITORY_NAME.git)
    ```
2.  Download and install the free version of [Power BI Desktop](https://powerbi.microsoft.com/desktop/).
3.  Open the `dash.pbix` file contained in the root folder.
4.  **How to Interact:** Since the project is opened in "Edit Mode" locally inside the Desktop client, **hold down the `Ctrl` key on your keyboard** while left-clicking any of the UI thumbnail buttons or timeline toggle buttons to watch the dashboard dynamically transform!
